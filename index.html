class MyClass {
public:
  MyClass();
  ~MyClass();
private:
  class Impl;
  std::unique_ptr<Impl> impl_;

第一次看到这种写法，很多人都会疑惑：

  • 为什么真正的数据成员不放在类里面？
  • 为什么要多定义一个Impl类？
  • 为什么析构函数要放到.cpp中实现？
  • 这样设计有什么好处？

实际上，这就是C++中非常经典的Pimpl(Pointer to Implementation)设计模式。本文将从设计思想、实现方式、优缺点以及使用场景几个方面详细介绍Pimpl。
